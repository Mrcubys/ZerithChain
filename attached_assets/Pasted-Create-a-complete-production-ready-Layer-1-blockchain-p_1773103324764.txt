## 2. Node Architecture & Blockchain Core

### Node Directory Structure

zerith-node
│
├── consensus       # ZPoS consensus & validator selection
├── blockchain      # Block storage & validation
├── mempool         # Pending transactions
├── networking      # P2P communication
├── validator       # Staking & block production
├── rpc             # API for wallets & dApps
├── storage         # Persistent blockchain data
└── crypto          # Hashing & signatures

### Core Data Structures

**Block Structure**
block_height
block_hash
previous_hash
timestamp
validator
transactions
state_root
signature

**Transaction Structure**
tx_hash
from_address
to_address
amount
gas_fee
nonce
timestamp
signature

**Address Format**

* Lowercase only, bech32
* Example: zth1xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
* Same format for mainnet & testnet

## 4. Global Validator Network

### Structure

* Total Validators: 100
* Regional Clusters: Americas, Europe, Asia, Africa
* Validator Requirements: CPU 8 cores, RAM 32GB, Storage 1TB SSD, Bandwidth 1Gbps
* Minimum Stake: 50,000 ZTH
* Block Time: 2s
* Slashing: Active for malicious/inactive validators
* Delegation: Token holders can delegate stake

### Validator Flow

Transaction -> Mempool -> Block proposer (ZPoS) -> Verification by validators -> Block finalization

### Monitoring & Redundancy

* Health check for all validators
* Failover for downtime
* Latency monitoring per region
