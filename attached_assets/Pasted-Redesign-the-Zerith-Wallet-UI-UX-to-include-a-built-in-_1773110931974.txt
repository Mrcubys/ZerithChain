Redesign the **Zerith Wallet UI/UX** to include a **built-in wallet browser** and move all staking functionality into the **Browser menu**, removing it from the Wallet tab. Ensure professional **OKX-style design**, white/light blue theme, and real ZTH QR code functionality.

---

### 1. Layout & Navigation

* **Bottom navigation bar (mobile) / Sidebar (desktop):**

  * **Wallet** – asset balances, portfolio, QR code
  * **Transaction History** – full list of user transactions
  * **Browser** – built-in wallet browser containing submenus: Explorer, Validator, Whitepaper, Stake
  * **Settings** – account, security, network options

* **Browser Submenus:**

  1. **Explorer** – Etherscan-like functionality (blocks, addresses, transactions)
  2. **Validator** – list of validators (empty if none)
  3. **Stake** – staking interface, rewards, delegation
  4. **Whitepaper** – embedded Zerith Chain whitepaper

---

### 2. Wallet Tab

* **Assets overview:** token balance and fiat equivalent
* **Portfolio summary:** total ZTH and fiat value
* **QR code section:**

  * Display your **real ZTH address** QR code
  * Scan **other ZTH addresses** to pre-fill Send form
* **Send / Receive buttons**
* No staking functionality here

---

### 3. Transaction History Tab

* Columns: Date/Time, Type (Send/Receive), Amount, Status
* Filters: Token, Status, Date Range
* Pagination & sorting
* SHA256 **Zerith-native tx hashes**
* Clickable hashes open Explorer view

---

### 4. Browser Tab (Built-in Wallet Browser)

* Functions as **wallet-integrated web browser** to interact with dApps, contracts, staking interfaces, and blockchain data

**Submenus:**

1. **Explorer:**

   * Search blocks, addresses, transactions
   * Address view: balance, total received/sent, transaction count
   * Transaction table: SHA256 tx hash, age, from, to, value, fee, status
   * Block view: height, timestamp, validator, transactions
   * Transaction view: hash, block, from/to, amount, fee, timestamp

2. **Validator:**

   * Table: validator name, stake, status, uptime
   * Empty state: “No validators yet”

3. **Stake:**

   * Stake your ZTH tokens
   * View current stake, rewards, and delegation options
   * Interface for delegation and unstaking

4. **Whitepaper:**

   * Full Zerith Chain whitepaper embedded
   * Table of contents for navigation

---

### 5. QR Code Integration

* Display **real ZTH address** QR code in Wallet tab
* Scan QR codes from others’ ZTH addresses to pre-fill **Send form**
* Validate scanned addresses (`zth` + 50 alphanumeric characters)

---
